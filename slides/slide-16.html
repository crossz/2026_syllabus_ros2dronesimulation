Service: Request-Response Model

Core Components

  • Service Client: Node that sends request
  • Service Server: Node that handles request
  • Request: Input data for the service
  • Response: Output data from the service

Synchronous Blocking

  • Blocking Call: Client waits for response
  • No Return Until: Service completes processing
  • Timeout: Can set wait timeout
  • Use Case: Commands that need result

Use Cases

  • Query: Get robot state/info
  • Control: Execute action commands
  • Configure: Change parameters
  • Trigger: One-time actions
